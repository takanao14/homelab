# Homepage

[Homepage](https://gethomepage.dev/) is a modern, fully static, fast, secure fully customizable application dashboard with integrations for over 100 services and translations into over 40 languages.

## Directory Structure

- `base/`: Common Kubernetes manifests and default configuration files.
  - `config/`: Homepage configuration files (YAML, CSS, JS).
- `overlays/`: Environment-specific overrides.
  - `dev/`: Development environment settings.
  - `homelab/`: Production (Homelab) environment settings.

## Deployment

This project uses Kustomize for deployment.

### Development Environment

```bash
kubectl apply -k overlays/dev
```

### Homelab (Production) Environment

```bash
kubectl apply -k overlays/homelab
```

## Configuration

Homepage is configured using the files in `base/config/`. These are mounted into the pod via a ConfigMap.

- `services.yaml`: List of services and integrations displayed on the dashboard.
- `widgets.yaml`: Dashboard widgets (e.g., system info, weather).
- `settings.yaml`: General dashboard settings (title, theme, etc.).
- `kubernetes.yaml`: Kubernetes integration settings.
- `proxmox.yaml`: Proxmox integration settings.

### Secrets and Environment Variables

To keep the configuration clean and secure, sensitive information (passwords, API keys) and environment-specific values (IPs, URLs) are injected via environment variables.

In the YAML configuration files, use the `{{HOMEPAGE_VAR_NAME}}` syntax. These variables are defined in the `.env` file within each overlay directory:

- `overlays/dev/.env`
- `overlays/homelab/.env`

These environment variables are automatically loaded into the Homepage container via a `ConfigMap` generated by Kustomize's `envs` field.

## Features

- **Service Monitoring**: Integrated monitoring for DNS (PowerDNS), Hypervisors (Proxmox), and Storage (TrueNAS).
- **Infrastructure Status**: Real-time status of networking elements and homelab services.
- **Security**: Sensitive data is abstracted into environment variables and not hardcoded in the base configuration.
- **Customization**: Custom CSS and JS for a personalized look and feel.
